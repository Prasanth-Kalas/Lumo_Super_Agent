## Contract Decision

Deepgram's current official token path for client-safe temporary credentials is `POST /v1/auth/grant`, which issues short-lived JWTs with `usage:write` for Listen/Speak/Read. This branch uses that current Auth Grant endpoint instead of minting project keys through a project-scoped management endpoint. The Lumo client contract remains stable: `POST /api/audio/deepgram-token` returns `{ token, expires_at }` and never exposes `LUMO_DEEPGRAM_API_KEY`.

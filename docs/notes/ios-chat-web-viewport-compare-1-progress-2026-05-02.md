## What shipped

### iPhone SE 3rd-gen simulator (375pt ≈ web's 380px)

iOS doesn't have an arbitrary-viewport sim — devices come at fixed
point widths. The closest stock-iPhone narrow form factor to web's
380px is the iPhone SE (3rd generation) at 375pt × 667pt (1× scale)
or 750 × 1334 (2× scale). The 5pt difference is well below the
threshold of perception for a parity comparison and matches what
was already done in IOS-MIRROR-WEB-1 for the drawer captures.

Sim creation (one-time):

```
xcrun simctl create "iPhone SE Lumo" \
  com.apple.CoreSimulator.SimDeviceType.iPhone-SE-3rd-generation \
  com.apple.CoreSimulator.SimRuntime.iOS-26-4
```

### What matches at 375pt vs the iPhone 17 capture

The IOS-COMPOSER-AND-DRAWER-SCREENS-1 captures used iPhone 17
(393pt). The 375pt capture from this lane:

- Same hero ("How can I help today?")
- Same single-rounded-composer (mic icon when empty, paperplane
  when populated — Lane 3's swap)
- Tighter horizontal padding (composer pill almost reaches the
  screen edge on iPhone SE; iPhone 17 has visible gutter)
- "Carrier" + WiFi icons on the SE status bar (vs iPhone 17's
  notch which renders without text)

These differences are expected — narrower form factor = tighter
composer + smaller status bar real estate. The chat surface itself
is byte-identical between viewports, which is the parity claim.

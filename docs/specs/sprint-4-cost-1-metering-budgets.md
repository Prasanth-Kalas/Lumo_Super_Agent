# Sprint 4 COST-1 — Cost Metering + Budgets + Alerts

**Status:** Design draft, written during Kalas-Cowork session 2026-04-28, pending Kalas seal.
**Author:** Claude coworker (Cowork session), reviewed by Kalas.
**Implements:** Phase 4 W4 deliverable per `docs/specs/phase-4-master.md`

**Precondition:** SDK-1 shipped, PERM-1 shipped, ADR-016 sealed.

---

## Goal

Make ADR-016 executable. Meter every invocation, enforce per-user and
per-agent ceilings, fall back to cheaper models when budgets get tight,
and surface cost reporting to the user across in-app, email, and CSV
export.

After COST-1 ships:
- Every agent invocation writes one `agent_cost_log` row.
- The orchestrator refuses dispatch if the user's daily/monthly cap
  would be exceeded; the connector dispatcher and Brain SDK gateway
  refuse mid-flight calls that would push past the per-agent ceiling.
- The Brain SDK falls back to a cheaper model when forecast spend
  exceeds remaining budget; the fallback is logged.
- Daily 7am-local digest emails fire if spend > $0.01 the prior day.
- Monthly summary emails fire on the 1st.
- Workspace → Settings → Budget renders for free, pro, and enterprise.

COST-1 is the cost contract layer. ADR-013 declares the manifest
ceiling, ADR-014 declares scope-level spending caps, ADR-016 enforces
the per-user budget on top — three independent layers, defence in
depth.

## What previous sprints already shipped

- **SDK-1** — request envelope carries `budgets.remaining_user_daily_usd`,
  `remaining_user_monthly_usd`, `max_cost_usd_per_invocation`.
  Response envelope carries `cost_actuals { total_usd, brain_calls_usd,
  model_tokens_input, model_tokens_output, model_tokens_cost_usd,
  connector_calls }`. SDK has the data structures; COST-1 makes them
  durable.
- **PERM-1** — `agent_scope_grants.constraints` carries the per-scope
  spending caps from ADR-014 §4.3. The connector dispatcher already
  refuses calls that exceed `up_to_per_invocation_usd` and
  `per_day_usd`. COST-1 layers the per-user budget on top.
- **Phase 3 RUNTIME-1** — cost forecasting via `lumo_forecast_metric`.
  COST-1's cost-routing fallback consumes RUNTIME-1's forecast.
- **Brain SDK** (Phase 3 SDK-1) — already telemeters per-call cost in
  the response payload. COST-1 reads this for the cost log row.

## What this sprint adds

Five workstreams.

1. **Migration 030 — cost log + budget tier**
   `agent_cost_log` and `user_budget_tier` tables per ADR-016 §2 and §3.

2. **Orchestrator + dispatcher integration**
   - Pre-flight budget check (refuse dispatch if
     `max_cost_usd_per_invocation > remaining_daily`).
   - Per-call running-total subtraction in connector dispatcher and
     Brain SDK gateway.
   - End-of-invocation cost log row write (one per `request_id`,
     UPDATE on retry).

3. **Brain SDK cost-routing fallback**
   When `forecast_with_primary_model > remaining_user_daily_usd`, the
   SDK falls back to the cheaper model in the table from ADR-016 §5.
   Fallback evidence written to `agent_cost_log.evidence`.

4. **Reporting surfaces**
   - `app/settings/budget/page.tsx` — tier, caps, spend, top-5 agents.
   - `app/settings/agents/[id]/page.tsx` — extend PERM-1 page with
     month-to-date cost, daily sparkline, per-capability breakdown,
     forecast for rest of month.
   - CSV export at `app/api/budget/export/route.ts`.

5. **Cron-driven email**
   - `app/api/cron/cost-daily-digest/route.ts` — runs hourly; per
     user_id, fires email at the user's local 7am if prior day spend
     > $0.01.
   - `app/api/cron/cost-monthly-summary/route.ts` — runs daily; fires
     on the 1st of the user's local month.
   - Free-tier upgrade nudge: > 80% of cap on > 3 days in 7-day
     window triggers an in-app banner + email.

## Orchestrator integration

```ts
// lib/orchestrator.ts (modified)

export async function dispatchAgentInvocation({ user_id, agent_id, ... }) {
  // 1. Pre-flight budget check
  const budget = await getUserBudget(user_id);                 // 5s cache
  const spent_today = await getSpentToday(user_id);            // 5s cache
  const spent_month = await getSpentMonth(user_id);            // 5s cache
  const remaining_daily = budget.daily_cap_usd - spent_today;
  const remaining_monthly = budget.monthly_cap_usd - spent_month;

  if (remaining_daily <= 0 || remaining_monthly <= 0) {
    await fireBudgetBreachAlertIfFirst(user_id);
    throw new AgentError("BUDGET_EXCEEDED", "user budget exceeded", false);
  }

  const manifest = await getManifest(agent_id);
  if (manifest.cost_model.max_cost_usd_per_invocation > remaining_daily) {
    // Refuse before agent runs
    throw new AgentError("BUDGET_EXCEEDED", "manifest ceiling exceeds remaining daily", false);
  }

  // 2. Build envelope with budget context
  const envelope = buildAgentInvocationEnvelope({
    ...,
    budgets: {
      remaining_user_daily_usd: remaining_daily,
      remaining_user_monthly_usd: remaining_monthly,
      max_cost_usd_per_invocation: manifest.cost_model.max_cost_usd_per_invocation,
    },
  });

  // 3. Initialize cost accumulator (in-memory per request_id)
  const accumulator = createCostAccumulator(envelope.request_id, manifest.cost_model.max_cost_usd_per_invocation);

  // 4. Dispatch (connector dispatcher and Brain SDK both consult accumulator)
  let response;
  try {
    response = await dispatch(envelope, accumulator);
  } finally {
    // 5. End-of-invocation cost log write (UPSERT on request_id)
    await writeCostLogRow({
      request_id: envelope.request_id,
      user_id, agent_id, agent_version: manifest.version,
      mission_id, mission_step_id,
      brain_calls_usd: accumulator.brain_usd,
      brain_call_count: accumulator.brain_call_count,
      model_tokens_input: accumulator.tokens_in,
      model_tokens_output: accumulator.tokens_out,
      model_tokens_cost_usd: accumulator.token_usd,
      connector_calls: accumulator.connector_call_count,
      connector_calls_usd: accumulator.connector_usd,
      total_usd: accumulator.total(),
      status: response?.status === "succeeded" ? "completed"
            : accumulator.aborted_budget ? "aborted_budget"
            : "aborted_error",
      evidence: accumulator.evidence(),
    });
  }
  return response;
}
```

The cost log write is async-fire-and-forget with a hard sync at
invocation end (per ADR-016 §9.2 mitigation).

## Connector + Brain SDK per-call subtraction

Both dispatchers consult the in-memory cost accumulator before each
call:

```ts
async function consumeBudget(accumulator, forecast_usd) {
  if (accumulator.total() + forecast_usd > accumulator.per_invocation_cap) {
    accumulator.aborted_budget = true;
    throw new AgentError("BUDGET_EXCEEDED", "would exceed per-invocation cap", false);
  }
  if (accumulator.total() + forecast_usd > accumulator.remaining_user_daily) {
    accumulator.aborted_budget = true;
    throw new AgentError("BUDGET_EXCEEDED", "would exceed remaining daily", false);
  }
  // proceed with call; on success, accumulator.add(actual_usd)
}
```

For Brain SDK calls specifically, the cost-routing fallback runs
before the budget check:

```ts
async function brainCall(tool, args, accumulator) {
  const forecast = await runtimeIntelligence.forecastCost(tool, args);
  if (forecast.usd > accumulator.remaining_user_daily * 0.5) {
    // Try fallback model
    const fallback = chooseFallback(tool);
    if (fallback) {
      const fallback_forecast = await runtimeIntelligence.forecastCost(fallback.tool, args);
      if (fallback_forecast.usd <= accumulator.remaining_user_daily) {
        accumulator.evidence.fallback_model_used = fallback.tool;
        return innerCall(fallback.tool, args, accumulator);
      }
    }
  }
  return innerCall(tool, args, accumulator);
}
```

Fallback table (from ADR-016 §5):
| Original | Fallback |
|---|---|
| `claude-opus-4` | `claude-haiku-4` |
| `gpt-5` | `gpt-5-mini` |
| `gemini-pro` | `gemini-flash` |

## Daily digest cron

`app/api/cron/cost-daily-digest/route.ts` runs every hour.

For each user_id where the user's local time is between 6:55am and
7:05am AND prior-day spend > $0.01:
- Compose email per ADR-016 §6.2 template.
- Send via the existing notification surface.
- Idempotent: keyed by (user_id, local_date) so re-running the cron
  the same hour doesn't double-send.

Free-tier nudge: if the user hit > 80% of daily cap on > 3 of last 7
days, append an upgrade nudge to the daily digest.

## Monthly summary cron

`app/api/cron/cost-monthly-summary/route.ts` runs daily at 00:00 UTC.

For each user_id where the user's local date is the 1st:
- Compose email per ADR-016 §6.3 template.
- Include "you saved $X via cheaper-model fallback" line.
- Include per-agent + per-capability breakdown.
- Include action recommendations (agents with high cost / low
  invocation count flagged).

## Acceptance

1. Migration 030 applied; default `user_budget_tier` row created on
   new profile signup.
2. Cost row written for every invocation; CI integration test runs a
   5-step mission with 3 agents, asserts exactly 3 rows with consistent
   `total_usd`.
3. Per-user budget enforcement test: a free user with $0.50/day cap
   is refused on the 11th invocation of a $0.05-each agent.

   `max_cost_usd_per_invocation: 0.04` that attempts $0.05 worth of

8. CSV export schema matches §"CSV export"; download works for a
   30-day window.

## Out of scope

- Pre-paid credits / Stripe Connect (Phase 5).
- Charity / non-profit tier (Phase 5).
- Per-tenant aggregation for Enterprise (Phase 5; data-modelled now
  via a follow-on `tenant_id` column in `user_budget_tier`).
- Brain-tool-specific cost ceilings independent of agents (Brain side
  has rate limits; revisit if a single tool dominates).
- "Continue for $X more" grace prompt UI (Phase 4.5).
- Bandit-driven model selection (Phase 4.5; cost-routing fallback in
  v1 is rule-based per ADR-016 §5).
